# v0.43.0 — 供应商/模型管理页重构

## 重构

### 供应商页（ProvidersTab）全面重构
- **合并 ProvidersTab + ModelsTab** 为以 Provider 为核心的单一管理页
- **左右双栏布局**：左栏 Provider 列表（OAuth + 内置预设 + 自定义），右栏 Provider 详情
- **统一 API 概览 API**：新增 `GET /api/providers/summary`，合并 providers.yaml + OAuth + favorites
- **内置供应商列表**：PROVIDER_PRESETS 全部展示在左栏，未配置的灰色显示
- **OAuth 白名单**：只暴露 minimax / openai-codex，其他 coding plan 不展示（避免封号）

### Provider 详情面板
- **统一视图**：已配置和未配置的 provider 使用完全相同的页面结构
- **API Key 直接输入**：无编辑模式切换，输入框 + 连接图标按钮
- **连接验证**：点击连接图标验证 API 连接，灰色=未验证，绿色=通过，红色=失败
- **未配置 preset**：API Key placeholder 显示提示文案，验证通过自动注册
- **API 类型下拉**：使用 SelectWidget 替代原生 select
- **删除确认弹窗**：防误操作

### 模型管理
- **已收藏模型列表**：在凭证区和添加模型之间，紧凑展示，限高滚动
- **模型编辑弹出卡片**：居中弹出 + 遮罩，支持编辑模型名称、上下文长度、最大输出
- **模型名 vs ID 分离**：displayName 存 overrides，列表显示 displayName（有的话）
- **添加模型下拉**：fixed 定位不被容器截断，搜索 + 自定义输入
- **添加自定义供应商**：弹出浮动卡片（360px），包含名称/URL/Key/API类型

### 聊天页 ModelSelector
- **按 provider 分组展示**：多 provider 时显示分组标题
- **当前模型强制出现**：即使不在 favorites 也显示在列表中

## 删除
- `ModelsTab.tsx` — 功能已合并到新 ProvidersTab
- SettingsNav 中 models tab 入口

## 新增文件
- `GET /api/providers/summary` (server/routes/providers.js)

## Store 变更
- 新增 `ProviderSummary` 类型、`providersSummary`、`selectedProviderId`
